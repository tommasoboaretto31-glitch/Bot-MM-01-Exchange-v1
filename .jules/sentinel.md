## 2025-02-19 - [Cross-Site WebSocket Hijacking in Dashboard]
**Vulnerability:** The local dashboard WebSocket endpoint (`/ws`) did not validate the `Origin` header. WebSockets are not restricted by CORS policies, meaning any malicious website could connect to the local bot's WebSocket server and silently stream sensitive data (like bot positions, balance, strategies, and logs) without triggering browser security protections.
**Learning:** Even internal/local dashboards running on `127.0.0.1` are vulnerable to cross-site attacks if an attacker can trick the user into visiting a malicious webpage. While REST endpoints may be protected by the lack of CORS headers, WebSocket connections require explicit `Origin` validation.
**Prevention:** Always validate the `Origin` header for WebSocket endpoints, especially in local servers. Ensure that `Origin` matches the expected `Host` or a tightly scoped whitelist of domains. Additionally, always use CORS middleware with strict allowed origins.
